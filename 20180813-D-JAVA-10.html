1.背景介绍

MongoDB最初是由MongoDB公司开发的，该公司当时(2007)被命名为10gen。该公司由前DoubleClick创始人和工程师，特别是Dwight Merriman，Kevin P. Ryan和Eliot Horowitz 创立。

起初，10gen想要建立一个开源平台作为服务。该公司希望其软件的所有组件都是完全开源的，但无法找到满足其需求的数据库。

10gen平台的平台名叫Babble，与Google App Engine类似。事实证明，Babble并没有一个很大的市场，但是Babble的用户和非用户同意，在这个平台上创建的10gen数据库非常出色，而且会很乐意自己使用。

虽然最初被称为“p”，数据库被正式命名为MongoDB，“Mongo”一词的缩写。鉴于输入10gen公司 已收到的有关MongoDB中，该公司决定最终在2009年废除Babble项目并自行发布MongoDB作为开源数据库平台。

到2012年，10gen被评为“华尔街日报”发表的“2012年下一个大事件”第9名，并在世界各地设有6个办事处。2013年，10gen公司更名为MongoDB，希望与它的流行的主要产品有紧密的联系。

总结起来就是：10gen这家公司想建立开源平台，结果没有找到满足自身需要的数据库，于是这家公司自己造了一个

2.1 mongoDB是什么？

MongoDB 是一个基于分布式文件存储的数据库。由 C++ 语言编写。旨在为 WEB 应用提供可扩展的高性能数据存储解决方案。

MongoDB 是一个介于关系数据库和非关系数据库之间的产品，是非关系数据库当中功能最丰富，最像关系数据库的。

MongoDB 将数据存储为一个文档，数据结构由键值(key=>value)对组成。MongoDB 文档类似于 JSON 对象。字段值可以包含其他文档，数组及文档数组。

MongoDB在幕后以二进制编码格式表示称为BSON的JSON文档。BSON扩展了JSON模型，以提供其他数据类型，有序字段，并有效地在不同语言中进行编码和解码。

2.2 mongoDB概念解析

文档是一组键值(key-value)对(即BSON)。MongoDB 的文档不需要设置相同的字段，并且相同的字段不需要相同的数据类型（可直接演示），这与关系型数据库有很大的区别，也是 MongoDB 非常突出的特点。

mysql采用table和结构化的sql语句来处理数据，在mysql中需要预先定义数据结构schema，并定义table中数据字段的关系。

mongoDB采用动态数据模型schema，这意味着不需要预先定义表的数据类型和字段名。

在mysql中，相关信息可以保存在不同的表中，通过join的形式来保持彼此关联。

mongoDB采用类JSON的documents来存储数据。在mongoDB的查询语句中，相关信息由于可以采用灵活的数据结构存储在一块，所以查询速率非常快。
