1.背景介绍

什么是JDBC？

JDBC（Java DataBase Connectivity即java数据库连接）是一种用于执行SQL语句的Java API，可以为多种关系数据库提供统一访问，它由一组用Java语言编写的类和接口组成。

JDBC说白了就是一套AIP接口,一套规范.然后各大数据库的驱动都实现并遵守了这套接口.

优点：运行期：快捷、高效

缺点：编辑器：代码量大、繁琐异常处理、不支持数据库跨平台

什么是JDBC Template

而多的这个template,就是模板,是Spring框架为我们提供的，所以JDBCTemplate就是Spring对JDBC的封装,通俗点说就是Spring对jdbc的封装的模板。

jdbcTemplate是spring框架中关于dao的模块，基本上是对原生jdbc访问数据库的二次封装，简化了jdbc的操作。

优点：运行期：高效、内嵌Spring框架中、支持基于AOP的声明式事务

缺点：必须于Spring框架结合在一起使用、不支持数据库跨平台、默认没有缓存

2.知识剖析

基本原理

首先讲JDBC肯定离不开DAO（Data Access Object）层，而JDBC的原理就是通过Connection对象获取数据库的连接，然后通过PreparedStatement对象处理SQL语句，再通过它的.setObject方法传入数据，最后通过方法.executeUpdate()和.executeQuery()执行更新，这就是JDBC的基本原理。

实现步骤

加载JDBC驱动程序

建立数据库连接Connection

创建执行SQL的语句Statement

处理执行结果ResultSet

释放资源

基本原理

JDBCTemplate组合了基类JdbcAccessor和接口类JdbcOperation。

在基类JdbcAccessor的设计中，对DataSource数据源进行管理和配置。

JdbcOperation接口中，定义了通过Jdbc操作数据库的基本操作方法，

模板方式使各个execute函数，组合了回调接口，各种Callback，query方法、update方法和batchupdate方法等是execute模板方法中逻辑的具体实现。

实现步骤

创建dataSource对象（其中包含连接数据库的信息）

通过dataSource创建JdbcTemplate对象

通过JdbcTemplate获得数据库数据

JDBCTemplate和JDBC的优缺点

jdbc

优点：运行期：快捷、高效

缺点：编辑器：代码量大、繁琐异常处理、不支持数据库跨平台

JDBCTemplate

优点：运行期：高效、内嵌Spring框架中、支持基于AOP的声明式事务

缺点：必须于Spring框架结合在一起使用、不支持数据库跨平台、默认没有缓存

PrepareStatement和Statement有什么区别

Statement每次执行SQL语句，相关数据库都要进行SQL语句编译

PrepareStatement是预编译，对批量处理大大提高效率，可以防止SQL注入攻击，防止数据库缓冲池溢出，代码的可读性，可维护性都更好
