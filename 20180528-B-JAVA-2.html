1.背景介绍

现在是数据运用广泛的时代，我们的数据库往往会存储很多信息。这样就面临着一个问题，当我们需要把数据库的数据呈现给客户的时候，不可能把所有数据都写在一个页面上，需要分页显示才行，这就需要进行分页查询和显示。

2.知识剖析

清理思路

1 创建页面类，存储页面信息；

2 根据页面信息进行数据库查询；

3 把查询到的数据呈现给客户；

确定数据量和页数

1 尽量做完spring和mybatis的整合之后，利用service来连接数据库

2 主要问题是根据数据总量来确定页面数

3 为了能尽快确定数据库的数据量，要给查询总量的字段建立索引，加快查询速度。根据数据总量确定totalUsers

4 select name from tb_test

5 totalPages = totalUsers % pageSize == 0 ? totalUsers / pageSize : totalUsers / pageSize + 1;

实现分页查询

1 分页查询的sql语句：select * from tb_test limit #{start},#{perPageUsers}

2 查询结果放在List集合里面，返回给页面

3.常见问题

页面跳转不对，数据连续性有问题

4.解决方案

limit #{start},#{perPageUsers}

1 起始位置是不包含的，比如start=1，那么是从数据库的第二条数据开始查询

2 页面信息没有进行传递，当前页一直都是初始值的1
