2.知识剖析

项目目录说明

复盘服务器上一个项目的目录分为三大块:

源码目录：通过SVN为中转站同步将我们的代码同步到服务器端

运行目录：项目运行的主要目录, 分为前端和后端

脚本目录: 一般用来存放自动部署脚本

4.解决方案

svn主要通过 commit 和 update 来完成提交本地数据和拉去svn上的数据

服务器上将配置分为三块, 一个是根据请求分发的servers.conf, 一个是负载均衡的upstreams.conf, 还有一个是配置动静分离的cat location_*.conf

编译是在服务端进行的, 使用maven clean install命令然后去模块下maven clean package 即可, 我们一般会加一个 -Dmaven.test.skip=true 来跳过测试部分. 部署jar包还是war包是要看你项目使用的框架, 我这里使用的spingboot可以很方便的运行jar包, 所以使用jar方式, 看wiki介绍war包部署服务器上使用的是resin, 这块了解的不多, 可以找师兄的部署脚本看看

部署脚本基本上就是把你手动执行的操作写到一个文件中运行, 可以一步一步写进脚本, 也可以编写函数运行.
